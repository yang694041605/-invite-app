# Excel导入功能说明

## 文件格式要求

为了成功导入Excel文件，您的文件必须满足以下格式要求：

### 1. 文件格式
- 支持的格式：.xlsx
- 不支持的格式：.xls, .csv, .txt等其他格式

### 2. 表头要求
Excel文件的第一行必须包含以下表头（不区分大小写，支持空格）：
- 期号
- 百位
- 十位
- 个位

表头示例：
```
期号	百位	十位	个位
```

### 3. 数据格式要求

#### 期号
- 格式：6位数字
- 示例：202301, 202302, ..., 2023365

#### 开奖号码
- 百位、十位、个位各为0-9的数字
- 每个位置只能有一个数字

### 4. 数据示例

```
期号	百位	十位	个位
202301	1	2	3
202302	4	5	6
202303	7	8	9
202304	0	1	2
```

## 常见问题及解决方案

### 1. "未找到有效的数据"错误

**可能原因：**
- 文件格式不正确（不是.xlsx格式）
- 表头不完整或名称不正确
- 数据格式不符合要求
- 文件为空或没有数据行

**解决方案：**
- 确保文件扩展名为.xlsx
- 检查表头是否包含"期号"、"百位"、"十位"、"个位"
- 验证期号是否为6位数字
- 验证开奖号码是否为0-9的数字
- 确保文件中有数据行（至少1行）

### 2. "文件读取失败"错误

**可能原因：**
- 文件损坏
- 文件权限问题
- 文件太大

**解决方案：**
- 尝试重新保存文件
- 检查文件权限
- 将大文件拆分为多个小文件

### 3. 部分数据被跳过

**可能原因：**
- 数据格式不正确
- 缺少必要的字段
- 数据类型不匹配

**解决方案：**
- 检查被跳过行的数据格式
- 确保所有必要字段都有值
- 验证数据类型是否数字

## 最佳实践

1. **数据准备**：
   - 确保数据按照要求的格式准备
   - 至少准备20期数据以获得更准确的预测结果

2. **文件检查**：
   - 在导入前检查文件格式和内容
   - 使用提供的示例文件作为参考

3. **分批导入**：
   - 对于大量数据，建议分批导入
   - 每次导入后验证数据的正确性

## 调试信息

为了帮助诊断导入问题，系统会在浏览器控制台.log中记录详细的调试信息，包括：
- 文件读取过程
- 表头验证结果
- 每一行数据的处理情况
- 错误信息和原因

如果您遇到遇到导入问题，请查看控制台日志以获取更多详细信息。
